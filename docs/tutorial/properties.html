Properties

Every component is managed by the properties. They allows you to control node's size, background, position, visibility and many more. Every node has its own set of properties, and all of them are listed in suitable documentation site.

Value

set

You can set value of the property in a simple way:label["text"] = "This text will be displayed by the label"

get

You are also able to get property's value:print("The label component is %d pixels wide" % label["width"])

Events

Properties can change their values using set and reset methods. The first one changes the value and invokes events, whereas the second one changes value without invoking them. What are the events?

Binding

Properties can be synchronized mutually. What does it mean? That means that if you change value of the property, all the bound properties change their own values immediately.slider_value = slider.property("value")
text_field_width = text_field.property("width")
text_field_width.bind_to(slider_value)
This code connects width of the TextField with Slider's value. When user changes the value by dragging thumb, width of the TextField changes immediately.
You can remove this connection using text_field_width.unbind() method.

Trigger

password = password_field.property("text")
password.add_trigger(validate)
This example code connects PasswordField text property with suitable action that is invoked whenever user changes content of the PasswordField (by writing something inside).
