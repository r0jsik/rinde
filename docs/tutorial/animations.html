Animations

An animation allows you to change property's value gradually. All you have to do is to invoke animate_to or animate_by method on the property object. Both of the functions accepts the same arguments: value, callback and speed, however they are different in action. The first one changes property's value to be the same as the value argument, whereas animate_by uses the same argument to change property by. The callback is a method invoked when animation finishes. The last argument specifies how fast the value changes.

Methods

animate_to

slider["range"] = 300
slider["value"] = 10
property = slider.properties["value"]
property.animate_to(250, speed=1)This animation changes Slider's value from 10 to 250. The animation takes about 4 seconds:
difference/(speed * fps) = (250 - 10)/(1 * 60) = 4

animate_by

slider["range"] = 300
slider["value"] = 220
property = slider.properties["value"]
property.animate_by(-60, callback)This animation changes Slider's value from 220 to 160 (if the initial value is 220, and the difference is -60, then the final value is 220 - 60 = 160). The callback is a reference to a method that will be executed when the animation ends. It takes about 0.5 second:difference/(default_speed * fps) = 60/(2 * 60) = 0.5
